SSHFling community package manifests

These files are generated from the release artifacts. Some ecosystems can install directly from these URLs; official/community repositories still require maintainer account submission, signing, and review.

Dependency ownership remains with the target operating system. Trust model: review the generated manifest, verify checksums, and confirm the package-manager trust root before installation.

  • Arch / AUR: PKGBUILD, .SRCINFO
  • Alpine: APKBUILD
  • FreeBSD Ports: security/sshfling
  • OpenBSD Ports: security/sshfling
  • pkgsrc: security/sshfling
  • Nix / NixOS: flake.nix
  • Guix: sshfling.scm
  • Void Linux: template
  • Gentoo: ebuild
  • Slackware: SlackBuild
  • openSUSE / OBS: spec
  • Snapcraft: snapcraft.yaml
  • Termux: build.sh
  • AppImage: AppImageBuilder.yml
